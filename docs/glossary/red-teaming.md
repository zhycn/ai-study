# 红队测试

## 概述

红队测试（Red Teaming）是一种主动的安全测试方法，通过模拟攻击者尝试发现和利用 AI 系统的安全漏洞。

## 为什么重要

- **漏洞发现**：主动发现系统弱点
- **防御提升**：针对性地提升防御
- **真实威胁**：模拟真实攻击场景
- **合规要求**：满足安全合规要求

## 测试维度

- **提示词注入**：测试注入攻击
- **拒绝服务**：测试系统稳定性
- **数据泄露**：测试敏感信息保护
- **偏见测试**：测试公平性问题

## 执行方式

- **内部团队**：内部安全团队执行
- **外部专家**：聘请外部安全专家
- **众包测试**：公开邀请测试
- **自动化测试**：使用自动化工具
