# Prompt Engineering

## 什么是提示词工程

提示词工程（Prompt Engineering）是**用自然语言编程**的艺术。它不是传统意义上的写代码，而是通过语言的结构、约束和示例来"编程"大语言模型（Large Language Model, LLM）的行为。

::: info 与词条的关系
[glossary/prompt-engineering](/glossary/prompt-engineering) 词条侧重解释"是什么"，本专题侧重指导"怎么做"。建议先阅读词条建立概念认知，再回到本专题学习实战技巧。
:::

### 为什么值得系统学习

- **成本效益**：相比微调（Fine-tuning），优化提示词的成本几乎为零，却往往能带来显著的效果提升
- **快速迭代**：无需重新训练模型，修改提示词即可即时验证效果
- **解锁能力**：许多模型的隐藏能力（如推理、规划、代码生成）只有通过特定的提示技巧才能激发
- **减少幻觉**：良好的提示词设计可以显著降低模型产生错误信息（Hallucination）的概率
- **可迁移性**：提示词技巧在不同模型之间具有较高的可迁移性，学习一次即可应用于多个平台

::: tip 经验法则
对于大多数应用场景，**先优化提示词，再考虑微调**。提示词工程能以最低成本解决 80% 以上的问题。
:::

## 内容导航

### 1. 基础篇

从零开始学习提示词工程的核心概念。

- 大语言模型工作原理简述
- **Zero-shot 提示**（零样本提示）与 **Few-shot 提示**（少样本提示）
- **角色扮演**（Role Prompting）技巧
- 常用提示词公式与模板

适合刚接触大模型、希望快速上手的新手。

### 2. 技巧篇

掌握让提示词更可靠的实用技巧。

- **结构化输出**：让模型按指定格式返回数据
- **分隔符**（Delimiter）：清晰区分指令与内容
- **负向提示**（Negative Prompting）：明确告诉模型不要做什么
- **上下文管理**：在有限的上下文窗口（Context Window）内高效组织信息
- **多模型适配**：针对 GPT、Claude、Gemini 等模型的差异化策略

适合已经写过提示词、但输出质量不稳定的用户。

### 3. 进阶篇

探索激发模型深层推理能力的高级模式。

- **思维链**（Chain of Thought, CoT）：引导模型逐步推理
- **思维树**（Tree of Thoughts, ToT）：探索多条推理路径
- **自我反思**（Self-Reflection）：让模型自行检查并修正输出
- **ReAct 模式**：推理与行动交替执行
- **提示词链**（Prompt Chaining）：将复杂任务拆分为多个提示词步骤
- **自一致性**（Self-Consistency）：多路径推理投票选出最优答案

适合需要处理复杂推理、规划任务的开发者。

### 4. 工程篇

将提示词从"能用"提升到"好用且可维护"。

- **版本管理**：用 Git 管理提示词变更历史
- **模板化**：将提示词抽象为可复用的模板
- **评估体系**：定义指标、构建测试集、自动化验证
- **A/B 测试**：科学对比不同提示词版本的效果
- **性能优化**：在效果与成本之间找到平衡点

适合将提示词集成到生产环境的工程师。

### 5. 安全篇

识别和防御提示词相关的安全风险。

- **提示词注入**（Prompt Injection）攻击原理与案例
- 防御策略：分隔符隔离、防御性指令、输出审查
- **幻觉防护**：降低模型编造信息的概率
- **敏感信息处理**：避免在提示词中暴露密钥和个人数据

适合关注生产安全的开发者。

### 6. 模式库

按场景分类的提示词模式速查手册。

- **文本处理**：摘要、翻译、改写、分类
- **代码辅助**：代码生成、审查、调试、解释
- **数据分析**：结构化提取、趋势分析、可视化建议
- **创意写作**：故事生成、文案撰写、头脑风暴
- **对话系统**：客服机器人、多轮对话管理
- **Agent 协作**：多智能体任务分配与协调

适合需要快速查找参考模式的开发者。
